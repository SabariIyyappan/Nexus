MEETING: Infrastructure Evaluation — Kafka
DATE: 2024-12-12
ATTENDEES: tom_wright, priya_sharma, devops_lead (external)
================================================================================

AGENDA: Should we adopt Kafka for event processing?

SUMMARY OF EVALUATION:
priya_sharma led a 2-week evaluation of Kafka for our async event processing needs.
Kafka would solve our synchronous messaging bottlenecks including the refund handler.

PROS:
- Solves async processing needs for payments pipeline
- Industry standard for high-throughput event streaming
- Would allow refund processing to become truly asynchronous (solving PERF-231)
- Decouples services, improves resilience

CONS:
- Significant operational complexity
- Requires Kafka expertise to run in production (cluster management, monitoring)
- No one on current team has Kafka production experience
- Training time: estimated 3-4 months to get team up to speed

DECISION: Do not adopt Kafka at this time. Primary blocker is operational expertise.
We have no one on the team who can own a Kafka cluster in production.
Document in INFRA-45 as rejected. Revisit if team expertise changes.

OWNER: tom_wright
STATUS: Rejected — revisit if expertise situation changes
